# 트리거 키워드 목록 (Trigger Keywords)

이 스킬이 **자동으로 활성화**되는 키워드와 사용 시나리오를 정리합니다.

---

## 핵심 트리거 키워드

다음 키워드가 사용자 요청에 포함되면 이 스킬을 **우선 적용**합니다:

### 1순위 (강력 트리거)

| 키워드 | 동의어/변형 |
|--------|-------------|
| `주간 보고` | 주간보고, 위클리 보고, weekly report |
| `임원 보고` | 임원보고, 경영진 보고, executive report |
| `캠페인 결과 요약` | 캠페인 요약, 캠페인 리포트 |
| `프로젝트 회의 요약` | 회의 요약, 미팅 요약, meeting summary |

### 2순위 (일반 트리거)

| 키워드 | 동의어/변형 |
|--------|-------------|
| `보고용으로 요약` | 보고용 요약, 보고 형식으로 |
| `보고서 작성` | 리포트 작성, report 작성 |
| `회의록 정리` | 회의록 요약, 미팅노트 정리 |
| `결과 보고` | 성과 보고, 실적 보고 |

### 3순위 (컨텍스트 트리거)

| 키워드 조합 | 설명 |
|-------------|------|
| `요약` + `팀장/임원/상사` | 상급자에게 보고 목적 |
| `정리` + `공유/전달/발표` | 다른 사람에게 전달 목적 |
| `요약` + `표/지표/수치` | 데이터 중심 요약 필요 |

## 사용 시나리오별 예시

### 📅 주간/월간 보고

```
"이번 주 마케팅 회의 내용을 주간 보고용으로 정리해 줘"
"월간 프로젝트 진행 상황을 임원 보고 형식으로 요약해 줘"
"지난주 Slack 대화 내용을 주간 보고서로 만들어 줘"
```

### 📊 캠페인/프로젝트 리포트

```
"이 마케팅 캠페인 결과를 요약 보고서로 만들어 줘"
"프로젝트 킥오프 회의 내용을 정리해 줘"
"A/B 테스트 결과를 보고서 형태로 정리해 줘"
```

### 📝 회의록 정리

```
"오늘 팀 회의 내용을 요약해 줘"
"이 회의록에서 액션 아이템만 뽑아줘"
"Teams 회의 녹취록을 보고서로 정리해 줘"
```

### 📧 이메일/메시지 요약

```
"이 이메일 스레드 내용을 요약해 줘"
"Slack 채널 대화에서 결정 사항만 정리해 줘"
"고객 피드백 이메일들을 요약 보고서로 만들어 줘"
```

## 트리거 우선순위 로직

```
IF 키워드 in [1순위 목록]:
    → 스킬 즉시 적용
ELIF 키워드 in [2순위 목록]:
    → 스킬 적용 권장
ELIF 키워드조합 in [3순위 조합]:
    → 스킬 적용 고려
ELSE:
    → 일반 요약 처리
```

## 비-트리거 케이스

다음 경우는 이 스킬이 **적합하지 않습니다**:

| 요청 유형 | 대안 |
|-----------|------|
| 단순 번역 | 일반 번역 처리 |
| 창작/작문 | 일반 작문 처리 |
| 코드 리뷰 | 코드 분석 처리 |
| 개인 메모 정리 | 일반 요약 처리 |
| 학술 논문 요약 | 학술 요약 형식 적용 |

## 커스텀 트리거 추가

팀에서 자주 사용하는 표현을 추가할 수 있습니다:

```markdown
## 우리 팀 커스텀 트리거

| 키워드 | 설명 | 추가일 |
|--------|------|--------|
| [키워드1] | [설명] | YYYY-MM-DD |
| [키워드2] | [설명] | YYYY-MM-DD |
```

### 예시

```markdown
## 우리 팀 커스텀 트리거

| 키워드 | 설명 | 추가일 |
|--------|------|--------|
| 스탠드업 정리 | 일일 스탠드업 미팅 요약 | 2026-01-15 |
| 스프린트 리뷰 | 스프린트 회고 요약 | 2026-01-15 |
| OKR 보고 | OKR 진행 상황 보고 | 2026-01-15 |
```
